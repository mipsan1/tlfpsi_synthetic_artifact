# TL-FPSI Synthetic Reference Artifact Bundle

This directory contains the deterministic, seedable plaintext reference
simulator used to populate Tables XI, XII and Figures 1, 2 of the paper
"Threshold-Gated Labeled Fuzzy PSI with Explicit Leakage for Contamination
Audits" (IEEE TDSC submission).

## Files

- `simulator.py`: core simulator. Implements the ideal counting semantics,
  SimHash-style band candidate generation, overflow / no-truncation behavior,
  Hamming re-check inside the active-2PC layer, and the per-(b,r) metrics
  reported in Table XII.
- `make_fig1.py`: regenerates `fig1.pdf` (the three-panel Figure 1 of the
  paper) from `summary.json`.
- `make_fig2.py`: regenerates `fig2.pdf` (the multi-item repeated-audit
  Figure 2 of the paper) and writes `fig2_data.json`.
- `summary.json`: aggregated per-(b,r) statistics over the 100 trials of the
  synthetic study. This is the input to Table XI and Table XII.
- `per_trial.jsonl`: 400 per-trial raw records (4 settings x 100 trials)
  that are the source of truth for any recomputation beyond the median and
  P95 reported in Table XI. Each line is a self-contained JSON object.
- `fig1.pdf`, `fig2.pdf`: regenerated figures.
- `fig2_data.json`: per-setting survival and confusion set statistics for
  Figure 2.
- `verify_fig2_8_12.py`: independent seed-sweep validation of the
  Figure 2 (b,r)=(8,12) b=tau SimHash regime. Runs R=200 fresh
  independent seeds with R_per_setting=1 and a full sweep of
  20 seeds x 200 trials to tighten the survival estimate of
  Table XII. Used to verify the production Fig 2 values.
- `MANIFEST.json`: SHA-256 of every other file in the bundle.

## Reproducing the bundle

```bash
python3 simulator.py .          # writes summary.json and per_trial.jsonl
python3 make_fig1.py summary.json fig1.pdf
python3 make_fig2.py fig2.pdf
python3 make_manifest.py        # writes MANIFEST.json
python3 verify_fig2_8_12.py     # independent seed-sweep validation
```

## Configuration (paper Section VII-C)

- n_corpus = 800
- m_bench = 240
- s_inject = 48
- tau = 8 (Hamming threshold)
- K_max = 12000 (candidate cap)
- n_trials = 100
- q_star = 0.9 (per-bit agreement of honest-input SimHash model)
- seed_base = 20260713

## Honest-input model

A near match is generated by sampling a Hamming distance
h ~ p(h | h <= tau) proportional to Bin(d, 1 - q_star) for h in {0..tau},
then flipping exactly h random bits of a chosen corpus fingerprint. The
chosen benchmark index is the target. This is the conditional form of the
honest-input SimHash model of Assumption (simhash) in the paper.

## Deployment caveat

The values reported in Tables XI, XII and Figure 1 are empirical estimates
on a specific (n, m, q_star, tau, b, r) fixture. A deployment that operates
at a different regime should re-run `simulator.py` with deployment-specific
seeds before reusing these values. Heavy-tailed, adversarial, or shifted
input distributions can violate the assumptions of the SimHash model, in
which case the per-pair marginal failure term delta_hp of Definition
(delta-hp) of the paper must be added to the analysis.
